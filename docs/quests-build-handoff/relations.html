What sits on top of what

Your instinct was right: quests as the top level were the restriction. Flip it — sessions are the substrate you drive; a quest is an optional contract that attaches.

quest owns sessions (top-down)  →  sessions are the drivers; quests attach to them (optional)

read it two ways
free session — YOU → cockpit → spawn a session → drive it on the layer. Stops there. (your `prefix+p` chat flow)
quest — context → quest → attaches to session(s) → loop drives on the layer → artifact → gates → done. (the walk-away lane)

the point — same sessions, same layer underneath both. The quest is a hat a session can wear, not a box it lives in.
