Why the details pane looks like that. A quest is one of your ~/.docs plans, so the pane just surfaces its frontmatter: id (the goal as docs-title), project / date / agent / status, the related tickets from docs-related, and the description from the body. The one thing your plans do not have is the Definition of done, which is the gates block the quest adds. Same data you already author, viewable in the terminal, and still the rich browser render with o. No parchment, no panel, no serif: one background, monospace, lines.
