0Overview & entity

A todo is a lightweight markdown note optionally attached to a target project. Its defining feature is escalation: one or more todos can be "spawned" into a brand-new agent session in the project they belong to, with the todo bodies pre-loaded as the session prompt.

Attributes

FieldTypeNotes
idstringStable id, e.g. td-1719900000. Used by CLI verbs & mutations.
contentmarkdown stringThe note body. Multi-line markdown. This is the required field.
project_idstring (optional)Same entity as tracker & artifacts — the git repo identity from repo.Resolve(cwd).Identity. May be empty (unassigned).
doneboolDrives the Active / Done tabs and the row checkbox.
created_at / updated_atRFC3339Timestamps, mirrors Artifact.AddedAt.
session_idstring (optional)Provenance only — which session authored it (from QUESTMASTER_SESSION). Not used for scoping.
Plan · model on artifacts, not sessions Todos are project-scoped / global, not session-scoped. Persist them in a single root-level registry (<state-root>/todos.json) exactly like artifacts.json, carrying an optional project_id. This is a 1:1 copy of the state.Artifact template. Do not require a session.

End-to-end shape (mirrors the artifact pipeline)

CLI / mutation writes todos.json→ fsnotify classify → tracker Change→ fullTracker rebuilds snapshot→ event pushed over socket→ RuntimeStore → Todo dock

Selection & multi-select

Artifacts are single-select. Todos need multi-select for batch spawn/delete. Keep a selectedTodoIDs: Set<String> plus a focusedTodoID cursor. space toggles done on the focused row; x or ⌘-click toggles set membership; ⇧↑/↓ extends. The blue-ringed empty checkbox in the mock = multi-selected; solid blue = done.

Filter on which tabs?

Artifacts only show the filter on the broad All tab. Todos have no "all" tab, so show the @project:/text filter on both Active and Done. The Core text search (case-insensitive contains over content + project) ports from ArtifactDisplayState.filteredArtifacts.

Selecting one or more todos and hitting s (or the Spawn action) opens the existing New Session modal, pre-filled with the project's path and the todo bodies as the prompt. Focus lands on the Title field — the only thing left to type. Choose an agent, hit ⏎.

Constraint · single project only Spawning from todos of different projects is not allowed. The spawn action is disabled (and the CLI errors) unless every selected todo shares one project_id. Todos with an empty project cannot be spawned (there's no path to open the session in) — surface this as a disabled state.

5CLI surface

Everything the UI can do, the CLI can do — so agents can keep deferred items. New parent command qm todo with a shared --session persistent flag, mirroring cmd/artifact.go. Verbs that the socket parses emit indented JSON via writeJSON; add --json where a text default is friendlier.

# create — project defaults to $QUESTMASTER_SESSION's project
qm todo add "Add --search flag to qm todo ls"
qm todo add "Regenerate serve goldens" --project ~/code/questmaster
qm todo add --file notes.md            # body from a file / stdin (-)

# list & search — scope by status, filter by project/text
qm todo ls                                # active by default, JSON
qm todo ls --scope done
qm todo ls --scope all --project ~/code/questmaster
qm todo ls --search "goldens"          # same contains-match as the UI filter

# edit / status / delete — by id (accepts multiple)
qm todo edit td-1719900000 --content "Updated body"
qm todo edit td-1719900000 --project ""      # clear project
qm todo done td-1719900000 td-1719900042    # mark done
qm todo reopen td-1719900000
qm todo rm td-1719900000

# spawn — one session in the todos' shared project, bodies become the prompt
qm todo spawn td-1719900000 td-1719900042 --title "todo-cleanup" --primary claude
# errors if the ids span more than one project, or any has no project
Plan · command factory cmd/todo.go: newTodoCmd(store, client) parent + subcommand factories newTodoAddCmd / LsCmd / EditCmd / DoneCmd / ReopenCmd / RemoveCmd / SpawnCmd. Register in cmd/root.go NewRootCmd via root.AddCommand(newTodoCmd(o.store, o.client)). Resolve project_id with the same resolveArtifactProjectID helper (repo.Resolve(m.Cwd).Identity). spawn delegates to session.NewService(...).Spawn(...) like cmd/spawn.go, joining bodies into the prompt; add a --from-app bool for the socket path.

6State & persistence

Copy the artifact registry model exactly (internal/state/hookstate.go).

  • New file internal/state/todo.go: Todo struct + status constants + CRUD.
  • Durable root registry <state-root>/todos.json (serialized as { "todos": [...] }), flock via sibling todos.json.lock — mirrors ArtifactsRegistryPath / withArtifactsRegistryLock.
  • CRUD with explicit-root ...At variants: LoadTodosAt, UpsertTodoAt (upsert by id), RemoveTodoAt, SetTodoDoneAt, FilterTodos(items, scope, projectID, query), SortedTodos — mirror UpsertArtifactAt … FilterArtifacts.
  • No per-session sidecar is required (todos aren't session-scoped); a single registry is enough.
// internal/state/todo.go
type Todo struct {
    ID        string `json:"id"`
    Content   string `json:"content"`
    ProjectID string `json:"project_id,omitempty"`
    Done      bool   `json:"done"`
    SessionID string `json:"session_id,omitempty"`
    CreatedAt string `json:"created_at"`
    UpdatedAt string `json:"updated_at"`
}

7Serve wire contract

8Swift app plan

Core (pure)

  • Core/Models/TodoModels.swift — TodoItem (id, content, projectID, done, timestamps), decoded from the snapshot.
  • Core/Todo/TodoDisplayState.swift — text search + status/project filtering + movedSelection (port from ArtifactDisplayState).
  • Core/Mutations/NewTodoLogic.swift — create-modal form model.
  • Core/Mutations/MutationRequests.swift — add todoAdd/Edit/Delete/Done builders + spawnFromTodos prompt assembly.
  • Core/Stores/RuntimeStore.swift — expose todos from the snapshot.
  • Matching XxxTests for each (custom runner in app/Tests).

App (I/O & views)

  • App/Todo/TodoDockView.swift, TodoDockPaneModel.swift, TodoDockPane.swift, TodoDockCoordinator.swift — port from App/Artifacts/*.
  • App/Todo/NewTodoSheet.swift + NewTodoRootView.swift — port from App/NewSession/*.
  • ShellTopBarViews.swift — add the checklist ChromeIconButton next to sidebar.right.
  • AppDelegate.swift — openNewTodo(), toggleTodoDock(), extend presentNewSession with initialPrompt + title focus.
  • Keymap.swift / MenuController.swift — ⌘T rebind + Todos menu.
Reminder · the app ships its own qm build-app.sh bundles the Go binary. Backend changes aren't live in the app until the bundle is rebuilt. Verify Swift via swift build --package-path app + the logic-test runner; leave interactive GUI checks to a human (no AppleScript-driven UI).
