0Overview & entity

Naming · "Quests", not todos These aren't strict todos — they're aspirational "things I want to do at some point", which fits the Questmaster brand. The entity is a Quest. The name is free: quest never appears in the codebase except inside "Questmaster", so there's no collision with the previously-ditched quests concept. Everything below uses quest / Quest / quests[] / qm quest.

A quest is a lightweight markdown note optionally attached to a target project. Its defining feature is escalation: one or more quests can be "spawned" into a brand-new agent session in the project they belong to, with the quest bodies pre-loaded as the session prompt.

Attributes

FieldTypeNotes
idstringStable id, e.g. qst-1719900000. Used by CLI verbs & mutations.
contentmarkdown stringThe note body. Multi-line markdown. Required.
project_idstring (optional)Same entity as tracker & artifacts — the git repo identity from repo.Resolve(cwd).Identity. May be empty → falls into the "No project" section.

created_at / updated_atRFC3339Timestamps, mirrors Artifact.AddedAt.
session_idstring (optional)Provenance only — which session authored it. Not used for scoping.
Plan · model on artifacts, not sessions Quests are project-scoped / global, not session-scoped. Persist them in a single root-level registry (<state-root>/quests.json) exactly like artifacts.json, carrying an optional project_id. A 1:1 copy of the state.Artifact template. Do not require a session.

Because the list is grouped by project, this is natural — but still enforce it: the spawn action is disabled (and the CLI errors) unless every selected quest shares one project_id. Quests in "No project" cannot be spawned (no path to open a session in) — surface as a disabled state.

6CLI surface

Everything the UI can do, the CLI can do — so agents can keep deferred items. New parent command qm quest with a shared --session persistent flag, mirroring cmd/artifact.go. Verbs the socket parses emit indented JSON via writeJSON.

# create — project defaults to $QUESTMASTER_SESSION's project
qm quest add "Add --search flag to qm quest ls"
qm quest add "Regenerate serve goldens" --project ~/code/questmaster
qm quest add --file notes.md              # body from file / stdin (-)

# list & search — scope by status, filter by project/text (grouped output)
qm quest ls                                 # active by default, JSON
qm quest ls --scope done
qm quest ls --scope all --project ~/code/questmaster
qm quest ls --search "goldens"            # same contains-match as the UI filter

# edit / status / delete — by id (accepts multiple)
qm quest edit qst-1719900000 --content "Updated body"
qm quest edit qst-1719900000 --project ""        # clear project
qm quest done qst-1719900000 qst-1719900042   # mark done
qm quest reopen qst-1719900000
qm quest rm qst-1719900000

# spawn — one session in the quests' shared project, bodies become the prompt
qm quest spawn qst-1719900000 qst-1719900042 --title "quest-cleanup" --primary claude
# errors if the ids span more than one project, or any has no project
Plan · command factory cmd/quest.go: newQuestCmd(store, client) parent + subcommand factories newQuestAddCmd / LsCmd / EditCmd / DoneCmd / ReopenCmd / RemoveCmd / SpawnCmd. Register in cmd/root.go NewRootCmd. Resolve project_id with the same resolveArtifactProjectID approach (repo.Resolve(m.Cwd).Identity). spawn delegates to session.NewService(...).Spawn(...) like cmd/spawn.go, joining bodies into the prompt; add a --from-app bool for the socket path.

7State & persistence

Copy the artifact registry model exactly (internal/state/hookstate.go).

  • New file internal/state/quest.go: Quest struct + CRUD.
  • Durable root registry <state-root>/quests.json (serialized as { "quests": [...] }), flock via sibling quests.json.lock — mirrors ArtifactsRegistryPath / withArtifactsRegistryLock.
  • CRUD with explicit-root ...At variants: LoadQuestsAt, UpsertQuestAt (upsert by id), RemoveQuestAt, SetQuestDoneAt, FilterQuests(items, scope, projectID, query), SortedQuests — mirror UpsertArtifactAt … FilterArtifacts.
  • No per-session sidecar required (quests aren't session-scoped); a single registry is enough.
// internal/state/quest.go
type Quest struct {
    ID        string `json:"id"`
    Content   string `json:"content"`
    ProjectID string `json:"project_id,omitempty"`
    Done      bool   `json:"done"`
    SessionID string `json:"session_id,omitempty"`
    CreatedAt string `json:"created_at"`
    UpdatedAt string `json:"updated_at"`
}
